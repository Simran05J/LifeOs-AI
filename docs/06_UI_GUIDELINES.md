# LifeOS AI - UI Guidelines

Version: 2.0

---

# Purpose

This document defines the UI and UX guidelines for the LifeOS AI application.

The goal is to build a modern, clean, and intuitive AI assistant interface.

---

# Design Theme

- Modern
- Minimal
- Clean
- Professional
- AI-focused

---

# Color Palette

Primary Color

- Purple (#6C63FF)

Background

- White (#FFFFFF)

Secondary Background

- Light Gray (#F5F5F5)

Text

- Dark Gray (#1F2937)

Success

- Green

Warning

- Orange

Error

- Red

---

# Typography

Recommended Font

- Poppins

Fallback

- Inter

Rules

- Large headings
- Clean body text
- Consistent font sizes

---

# Primary Screens

- Splash Screen
- Login
- Register
- Dashboard
- Chat Screen
- Voice Assistant
- Profile
- Settings

---

# Dashboard

The Dashboard should display:

- Welcome message
- AI Assistant Card
- Quick Actions
- Recent Conversations
- Planner Summary
- Reminder Summary
- Expense Summary

---

# Chat Screen

The Chat Screen should include:

- Chat messages
- Voice button
- Text input
- Send button
- Typing indicator
- Loading animation

---

# Voice Assistant

The Voice Assistant should support:

- Start Listening
- Stop Listening
- Speaking Indicator
- Voice Response

---

# Navigation

Use a simple navigation system.

Suggested Navigation:

- Home
- Chat
- Planner
- Profile

---

# Buttons

Buttons should have:

- Rounded corners
- Consistent padding
- Hover effects
- Disabled state
- Loading state

---

# Cards

Cards should be used for:

- Planner
- Expenses
- Trips
- Wellness
- AI Suggestions

---

# Animations

Keep animations subtle.

Recommended:

- Fade In
- Slide Up
- Loading Spinner

Avoid excessive animations.

---

# Responsive Design

The application should work on:

- Desktop
- Laptop
- Tablet
- Mobile Browser

---

# Error Handling

Display user-friendly messages.

Examples:

- Network Error
- Authentication Failed
- AI Service Unavailable

---

# Accessibility

- High contrast
- Readable font sizes
- Keyboard navigation
- Screen reader support where possible

---

# UI Rules

- Keep the interface uncluttered.
- Use consistent spacing.
- Do not overload the user with information.
- Focus on conversation-first interactions.
- Maintain the same design language across all screens.

---

# Goal

Create an elegant, responsive, and user-friendly interface that makes interacting with multiple AI agents feel simple and natural.
